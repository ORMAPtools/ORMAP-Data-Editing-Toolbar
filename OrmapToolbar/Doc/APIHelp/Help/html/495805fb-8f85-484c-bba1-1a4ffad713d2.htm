Application Architecture

[This is preliminary documentation and is subject to change.]

The ORMAP Taxlot Editing Toolbar application architecture is summarized in the Class Diagram (ClassDiagram.cd ). Each component of the architecture is expressed there.

This section explains the general architecture patterns used, including the application tiers, class types, and their relationships.

The classes in the application are organized into three tiers following a variation of the model-view-controler design pattern. In this variation, the view layer is implemented with very shallow graphical user interface classes with little or no behavior. The controler tier is divided into (a) interface-related classes which implement the required interfaces to extend ArcMap and handle all user-initiated events and (b) utility classes which provide support functions for the interface-related classes. The model tier is essentially provided by ArcObjects and is refered to here as the "Data Access Tier". The controler tier is refered to as the "Behavior Tier". The view tier is refered to as the "Interface Tier".

Interface Tier

The user interface tier is implemented with very shallow graphical user interface classes with little or no behavior.

Class-Linked Forms

The user interface consists of a set of classes (primarily Forms) each of which is paired with a controler class which implements the required interfaces to extend ArcMap and handle all user-initiated events.

Properties Form
Settings Form
Combine Taxlots Form
Locate Feature Dockable Window and User Control
Add Arrows Form
Dimension Arrows Form

Other Forms

These form classes are unusual in that the do host their own behavior. This approach is used because these classes are relatively independant of other features of the application, and do not require data access or other complex behaviors.

Help Form
About Form

Behavior Tier

The classes in this tier implement the application behavior.

Core Functionality

These classes provide the core functionality of the application. They implement the ArcObjects interfaces which allow them to extend the ESRI ArcMap user interface with additional toolbars and tools and to extend its Editor Extension behavior, providing an additional layer of behavior in response to edit events.

Editor Extension Class
Toolbar Class

User Configuration

These classes implement the interfaces and .NET framework members to provide access to the editor extensions properties and the application's custom user settings.

Property Page Class
Settings Accessor Class

Commands

These classes implement the ArcObjects ICommand interface which allows them to appear as commands on a toolbar or menu. All but the first have a form class associated with them.

Auto-Update-All Toggle Command
Combine Taxlots Command
Edit Map Index Command
Locate Feature Command (and Dockable Window)

Tools

These classes implement the ArcObjects ICommand and ITool interfaces which allow them to appear as tool commands on a toolbar or menu. Each has a main form class associated with it.

Add Arrows Tool
Taxlot Assignment Tool

Helper Classes

These classes are essentially utility classes which provide centralization for support functions for the interface-related classes.

Utilities Class (singleton)
String Utilities Class (singleton)
Data Monitor Class (singleton)
ORMapNum Class (used by EditMapIndex)
CatalogFileDialog Class (interface form provided by ArcObjects)

Data Access Tier

The data access tier is provided by the ArcObjects data components. These effectively abstract direct data manipulations and coordinate dynamic feature behavior.

Miscellaneous Classes and Components

The classes in this category play a support role. Either they are automatically generated by Visual Studio, are static resource files, or are part of the installer configuration.

Settings Designers
